1. Arquitectura y Esquema de Datos (Database Schema)

La base de datos centralizada se almacena en memoria caché de servidor y se sincroniza en la nube. A continuación se presentan las interfaces TypeScript de las entidades principales:

// 1. Evento
interface Event {
  id: string;                  // Prefijo 'evt_' + timestamp
  shortCode: string;           // Código de invitación corto (ej: 'house')
  title: string;               // Nombre del evento
  description: string;         // Descripción detallada
  date: string;                // Fecha (YYYY-MM-DD)
  time: string;                // Hora (HH:MM AM/PM)
  location: string;            // Ubicación física
  category: string;            // 'Hacker House' | 'Workshop' | 'Meetup'
  expectedAttendance: number;  // Aforo
  actualAttendance: number;    // Acreditados reales
  image: string;               // Portada del evento
  sponsors: Sponsor[];         // Patrocinadores asociados
  activities: Activity[];      // Misiones y agenda del evento
  startDate?: string;
  startTime?: string;
  endDate?: string;
  endTime?: string;
  ticketPrice?: string;
  timezone?: string;
}

// 2. Asistente (Usuario)
interface Attendee {
  id: string;                  // Prefijo 'at_' + timestamp
  name: string;
  email: string;
  walletAddress: string;
  walletType: 'Privy' | 'MetaMask' | 'Phantom' | 'Coinbase';
  joinedAt: string;
  points: number;              // XP acumulados
  completedActivities: string[];
  badges: UnlockedBadge[];     // Insignias NFT dinámicas
  checkedIn: boolean;
  calendarSynced: boolean;
  registeredEvents: string[];
  registeredActivities: string[];
}

// 3. Actividad / Misión
interface Activity {
  id: string;
  title: string;
  description: string;
  points: number;              // Puntos de XP
  type: 'CheckIn' | 'Keynote' | 'Workshop' | 'Hackathon' | 'SponsorVisit' | 'Feedback';
  required: boolean;
  sponsorId?: string;
  feedbacks?: Feedback[];
}

2. Catálogo de APIs (Endpoints)

Todas las rutas son relativas a la URL base de la aplicación y utilizan la cabecera Content-Type: application/json.

Método Endpoint Payload / Parámetros Descripción
GET /api/events Ninguno Obtiene la lista completa de todos los eventos creados en la base de datos.
POST /api/events Campos del evento en JSON Crea un nuevo evento personalizado de alta fidelidad con código de invitación corto.
GET /api/events/by-code/:code Código corto (ej: house) Resuelve y retorna los detalles del evento asociado al código de invitación.
POST /api/attendees/onboard name, email, walletAddress Crea el perfil del asistente a través de Privy y le otorga su insignia Genesis.
POST /api/attendees/:id/register-event eventId Registra el RSVP del asistente al evento para permitirle misiones.
POST /api/attendees/:id/activities/complete activityId, eventId Marca la misión como completada, suma XP y actualiza metadatos de su NFT.
POST /api/events/:id/activities/:actId/feedback rating, comment, attendeeName Guarda la opinión sobre una actividad y premia al participante con +50 XP.

3. UX, Flujos Clave y Lineamientos para el Equipo de Diseño

A. Acreditación Directa y RSVP

Al entrar desde un enlace de invitación tipo Luma (/invite/codigo), la tarjeta principal de RSVP debe ser el foco de atención primario. Si el participante aún no se ha registrado, se despliega el modal de Privy simplificado (ingreso con email directo). Una vez validada su cuenta, el botón se convierte en "Acreditado" y se le asigna su credencial virtual en el acto.

B. Misiones IRL (Códigos QR en stands)

Las misiones presenciales se validan mediante la lectura de códigos QR instalados físicamente en el evento. El flujo de UX debe desplegar el modal del escáner con crosshairs de cámara y la simulación del visor. Es indispensable que el modal conserve el diseño de degradados oscuros e incluya un campo alternativo de código manual por si la cámara presenta fallos de hardware.

C. Paleta Cromática y Tokens

El color primario para botones e interacción es el Indigo (#4f46e5) y el Violeta (#a855f7). El fondo siempre debe ser un gris oscuro mate o negro absoluto (#09090b). Los bordes deben ser delgados y con baja opacidad (rgba(255, 255, 255, 0.08)) para lograr un estilo traslúcido y moderno de Glassmorphism.
